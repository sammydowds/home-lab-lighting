Hue Motion Lighting Project

A Raspberry Pi-based automation system that uses a motion sensor to control Philips Hue lights via the Hue Bridge API.

Overview

This project detects motion using a PIR sensor connected to a Raspberry Pi and automatically turns Hue lights on. If no motion is detected for a configured period, the lights are turned off.

Features

  • Motion detection using GPIO PIR sensor
  • Automatic light activation via Hue Bridge API
  • Configurable timeout for inactivity
  • Environment-based configuration
  • Designed for headless Raspberry Pi operation

Architecture

This project will run a service via systemd on the pi. I created scripts to setup a few things automagically: 1) find the Hue Bridge IP, 2) register the device (to recieve a username), 3) store those in a file for future use, 4) create a group of lights that will be controlled by the Pi (note: the lights cannot currently belong to a group or room), and 5) setup a systemd service and run

  • Motion Sensor → GPIO input (gpiozero)
  • Controller → Python service loop
  • Hue API Layer → Custom Hue class wrapper
  • Bridge → Philips Hue Bridge local network API

Setup

Note: I enabled SSH access to my pi, which simplified interfacing with it.

1. Clone the repository to the pi

2. Run setup_hue_env.py

This will create a local .hue_env file in your /home/pi and will configure a group of lights on the network while also registering your device with the Hue Bridge.

    chmod +x setup_hue_env.py
    python3 setup_hue_env.py

3. Run start-service.sh

This will create a systemd service, enable it to run after restart, and start the service.

    chmod +x start-service.sh && ./start_service.sh

4. [NOTE] To stop and remove the service run remove-service.sh

    chmod +x remove-service.sh && ./remove_service.sh

How It Works

  1. PIR sensor detects motion
  2. Python script triggers Hue API call
  3. Lights turn ON
  4. Timer resets on each motion event
  5. If no motion is detected → lights turn OFF after timeout

Troubleshooting

  • Bridge not found: Ensure device is on same LAN
  • Connection errors: Check IP and username env variables
  • GPIO not triggering: Verify wiring and pin configuration
